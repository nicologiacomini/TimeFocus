## 🗄️ Database — Recreate / Reset (Local Development)

**⚠️ Warning:** These steps are for your local SQLite database (typically `DATABASE_URL="file:./dev.db"`). Do not run these in production.

### The Quick Way (Recommended)

Our helper script automatically backs up your existing database, installs dependencies, deletes the old DB, and sets up a fresh Prisma schema and client.

Run from the project root:

```bash
bun run db:recreate  # or: npm run db:recreate
```

### Alternative Methods

* **Manual Reset:** If you prefer running the steps yourself (make sure to back up `dev.db` first if needed):
```bash
rm -f dev.db prisma/dev.db
bun run db:push && bun run db:generate
```

* **Using Migrations:** If your project uses Prisma migrations, this will interactively drop data and reapply them:
```bash
bun run db:reset  # or: npx prisma migrate reset
```

### Troubleshooting

* **Database created in the wrong place?** Check that your `.env` or `.env.local` contains `DATABASE_URL="file:./dev.db"`.
* **Prisma import errors?** The client might need to be regenerated manually: `bun run db:generate`.
